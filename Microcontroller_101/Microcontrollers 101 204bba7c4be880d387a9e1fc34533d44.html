ATmega328p:

Paul recommended to learn Bare Metal on this piece of Silicon, the architecture itself it much simpler compared to anything on market, also the reason I actually picked this one is because it was available in the lab, I scraped it from ArduinoUNO

void setup() {
  long i = 0; //initializing 'i
  volatile byte* dir_reg = 0x24; // deaclaring a pointer dir_reg, that points to register 0x24
  //if you initialzed something and didnot used it, the complier gets rid of it
  //'volatile" tell the compliler not to get rid of this NO MATTER WHAT
  * dir_reg = 32; //00100000 sets PB5 as output, where PB5 means Port B, Pin 5
}

void loop() {
  volatile byte* led_port = 0x25;

  for (long i = 0; i < 1000000; i++) { * led_port = 32; }
  for (long i = 0; i < 1000000; i++) { * led_port = 0; }                               

}

This is the code I wrote in ArduinoIDE to make the LED Blink, there are probably better ways to implement this logic but anyways this is how it is,

Firstly we need to know where the User LED is placed on the microcontroller, for this you can either read the documentation for the Microcontroller(not the processor) or do what i did, Search “Where is the User LED on XYZ_microcontroller” in Perplexity, it results: PB5(for Arduino UNO)

Once we know the location of the LED, we have to mention this pin need to act like a Output pin, for that there are Data Direction Registers, so for Port B the register is 0x24, and the 5 bit of that register corresponds to LED, so we need to set 5th bit of 0x24 to 1 ( 1= Output, 0= Input)

Now we are ready to play with the LED, whenever the bit corresponding to the led goes ‘1’ it turns itself On and the corresponding register to this is 0x25

What PB5 means?

As it turns out, Pin address are different for the microcontroller and the processor, Pin 13 on the microcontroller would correspond to something like Port B Pin 5 on microcontroller abbreviated to PB5 there are sections of pins being labeled A,B,C,D and each section can have 8-32 bits depending on what kind of processor it is, in our case it is a 8bit processor
